# 功能名称

## 用户需求

- 用户希望完成什么。
- 当前痛点是什么。
- 成功后用户会看到什么变化。

## 功能逻辑

- 入口在哪里。
- 用户操作流程是什么。
- 关键状态如何变化。

## 架构设计

- 涉及哪些模块。
- 主要数据从哪里来，写到哪里。
- 与已有能力的边界是什么。

## 数据与存储

- 是否涉及 MySQL 表。
- 是否涉及 `AIstudyPublicData` 本地文件。
- 是否需要索引、去重、清理或快照保留策略。

## 错误与降级

- 哪些情况会失败。
- 用户看到的人话提示是什么。
- 后端记录哪些错误码和技术信息。
- 外部依赖不可用时如何降级。

## 验证记录

- 跑过哪些命令。
- 哪些场景人工验证过。
- 哪些风险还没有完全验证。

## 后续边界

- 后面扩展时不能破坏什么。
- 哪些事项必须先补架构或索引。
