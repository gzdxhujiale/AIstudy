# 数学字符与公式输入模块

## 定位

数学字符与公式输入要作为独立模块长期维护，不隶属于教材模块，也不隶属于 Word 文档模块。教材笔记、节点 Word 文档、后续题库解析和文档显示优化都只能调用它提供的输入、模板和渲染能力。

当前教材笔记里接入的 `Σ` 面板只是第一版落点。数学粘贴清洗已经抽到独立目录，后续字符面板、键入规则和公式层继续在该模块内扩展：

```text
src/renderer/features/mathInput
```

## 核心目标

- 能输入，也必须能保存和显示。
- `D_f`、`R_f`、`x^2` 这类内容不能只保存成普通字符串，应保存为富文本结构。
- 上标、下标优先使用 `canvas-editor` 原生 `superscript` / `subscript` 元素。
- 常用数学符号可以先作为内置静态字符表提供。
- 用户最近使用、收藏、自定义字符、模板排序后续走 DB-first StorageProvider，本地 JSON 只做兜底缓存。

## 第一阶段范围

- 教材笔记工具栏提供上标、下标按钮。
- 教材笔记工具栏提供 `Σ` 数学字符面板。
- 面板支持常用模板：`D_f`、`R_f`、`f(x)`、`x^2`、`x^-1`。
- 面板支持常用字符：集合、逻辑、箭头、比较、希腊字母等。
- 插入结果进入 canvas-editor snapshot，并通过现有教材笔记 MySQL 链路保存。
- 从 ChatGPT、KaTeX、MathML、HTML 或纯文本复制的数学内容，先进入共享粘贴解析层，再插入 canvas-editor snapshot。
- 粘贴解析至少稳定处理 `→`、`↦`、`⊂`、`⊆`、`∈`、`∉`、`ℝ`、`∞`、`≠`、`D_f`、`R_f`、`x^2` 和 `[0,+∞)`。

## 模块边界

- 不做假公式图，不把公式渲染成截图或纯 HTML 展示。
- 不依赖剪贴板作为唯一输入方式；但剪贴板进入编辑器时必须走统一清洗和结构化保留策略。
- 不把字符库数据散落到教材、Word、考试等多个组件中。
- 不在 UI 暴露技术解释、存储结构、快捷语法说明。
- 不把完整公式编辑器强行塞进第一阶段；后续确实需要复杂公式时，再评估 MathLive / KaTeX 作为独立公式层。

## 后续扩展

- 抽出共享 `MathCharacterPalette`。
- 增加搜索、最近使用、收藏和自定义字符。
- 增加键入规则，例如 `\R`、`\in`、`\ne`、`\to`。
- 增加模板编辑，例如 `D_{}`、`R_{}`、`f({})`、`{x | x ≠ 0}`。
- 对接 Word 文档编辑器工具栏，保持教材笔记和节点文档一致。
- 文档显示体系做大优化时，要统一检查 canvas-editor snapshot、教材笔记渲染、Word 文档渲染和导出效果，不能只优化输入面板。

## 显示优化约束

后续文档显示可能经历一次较大的排版优化。优化时必须保证：

- 已保存的 `superscript` / `subscript` 元素继续兼容。
- `D_f`、`R_f`、`x^2` 等旧笔记重开后显示不退化。
- 教材笔记载入 Word 文档后，上下标结构不丢失。
- 如果接入公式渲染库，公式数据必须有可迁移的结构化快照，不允许只保存渲染后的 HTML 或图片。
