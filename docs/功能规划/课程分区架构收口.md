# 课程分区架构收口

## 目标

课程和分区是知识库入口层，必须先保证结构稳定，再继续做拖拽、搜索、导出、复习卡、AI 总结等扩展。

本约束解决两个问题：

- 防止 UI、导图、文档、数据库互相耦合，避免一处功能改动引发另一处功能回归。
- 防止无效全量保存、重复快照、冗余镜像导致数据和内存膨胀。

## 分层归属

### 应用壳层

入口：`src/renderer/main.tsx`

职责：

- 持有当前页面和当前课程 id。
- 组合左侧课程分区、导图工作台、右侧目录。
- 打开和关闭课程弹窗。

禁止：

- 禁止继续在 `main.tsx` 堆课程分区搜索、折叠、移动菜单、内联编辑等 UI 状态。
- 禁止在应用壳里拼装课程或分区数据库结构。

### 课程功能 UI 层

入口：`src/renderer/features/course/`

职责：

- 呈现 Notion 式左侧树状分区。
- 管理搜索、移动菜单、内联新建分区、内联重命名分区等纯 UI 状态。
- 通过课程服务调用业务命令，不直接保存整包数据。

禁止：

- 禁止读取或修改思维导图编辑器实例。
- 禁止访问 MySQL 或拼 SQL。
- 禁止把 UI 临时状态写入课程数据，除非它是明确业务字段，例如分区折叠状态。

### 主进程服务层

入口：`electron/main.ts` 中课程服务函数，后续可独立拆到 `electron/services/course/`。

职责：

- Electron main 进程独占 MySQL 连接、迁移、事务和本地镜像。
- 对 renderer 暴露命令式 IPC。
- 每个业务动作只改受影响的行。

命令接口：

```text
courses:load
courses:create
courses:rename
courses:move
courses:reorder
courses:delete
courses:select

course-sections:create
course-sections:rename
course-sections:toggle
course-sections:reorder
course-sections:delete
```

兼容接口：

```text
courses:save
```

`courses:save` 只作为旧接口兼容保留，新功能不得继续依赖它做整包保存。

## 数据库职责

### `knowledge_sections`

保存课程分区轻量索引：

- `id`
- `name`
- `sort_order`
- `collapsed`
- `created_at`
- `updated_at`
- `deleted_at`

### `course_management_courses`

保存课程轻量索引：

- `id`
- `name`
- `description`
- `section_id`
- `sort_order`
- `created_at`
- `updated_at`
- `deleted_at`

课程表不得保存导图快照、Word 内容、资产内容或 AI 结果。

## 存储策略

- 新增、重命名、移动、排序、删除课程必须走命令式 IPC。
- 新增、重命名、折叠、排序、删除分区必须走命令式 IPC。
- 删除课程和分区采用软删除，避免误删和后续联动恢复困难。
- 删除分区时只软删除分区，分区内课程事务性移回「未分区」。
- 本地 `courses.json` 只作为轻量镜像和 MySQL 失败兜底，不是第二套事实源，也不是纯净发行版的初始数据源。
- 分区、课程排序、折叠状态等入口索引原则上属于数据库正式数据；纯净安装包不得携带开发机或验证机的本地分区/课程镜像。
- UI 切换课程只保存当前课程 id，不触发整包课程列表保存。

## MySQL 失败与恢复策略

- `courses:load` 优先连接 MySQL；连接失败时读取本地 `courses.json`，保证课程侧栏可用。
- 纯净新装场景下，如果安装包内没有真实用户运行数据，`courses:load` 的本地兜底应为空镜像；不得因为打包目录残留 `courses.json` 而显示旧分区。
- 安装后应优先自动检索公开版固定数据库配置或 AIstudy 管理的本机数据库服务并建立连接；当前已支持 `%ProgramData%\AIstudy\mysql\my.ini` 发现固定 `aistudy_public`，并在本机端口未就绪时短启动 `AIstudyMySQL` 服务。连接失败时，界面必须明确当前是本机镜像/本机模式。
- 课程和分区的新增、重命名、移动、排序、删除、折叠命令如果写 MySQL 失败，会先写入本地 `courses.json`，再追加一条轻量操作到 `course-pending-operations.json`。
- `course-pending-operations.json` 只保存课程/分区索引操作，不保存导图快照、Word 内容、资产内容或 AI 结果。
- MySQL 恢复后，下一次 `courses:load` 会先按顺序重放 pending 操作；全部成功后清空 pending 文件。
- 某条 pending 操作重放失败时，保留该操作和后续操作，记录 `retryCount` 与 `lastError`，继续回退本地镜像，避免半同步状态继续扩散。
- `courses.json` 和 `course-pending-operations.json` 必须原子写入；遇到历史损坏 JSON 时隔离为 `.corrupt-*.json`，应用继续启动。
- 当前选中课程 id 仍然只属于本地 UI 偏好，不写入 MySQL，也不进入 pending 队列。

## 拖拽排序准入

拖拽只能调用以下命令，不允许直接改 UI 列表后再整包保存：

```text
courses:move
courses:reorder
course-sections:reorder
```

- 同分区课程排序、跨分区课程排序、分区排序都必须事务性批量更新受影响行。
- 排序失败时只能进入课程 pending 队列，不得把导图、Word、资产内容放进 pending。
- UI 可以先展示本次命令返回的新列表，但排序事实仍以后端命令返回为准。
